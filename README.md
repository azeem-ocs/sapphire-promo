# Sapphire Studios — 20s Promo (Hyperframes + Claude Code)

A 20-second, 9:16 vertical promotional video for **Sapphire Studios** — a performance creative agency (UGC, PGC, AIGC, CGI, animations, and data-backed high-converting ads). Built entirely in HTML/CSS + GSAP and rendered to MP4 through [HeyGen Hyperframes](https://github.com/heygen-com/hyperframes).

**Final output:** [`video.mp4`](./video.mp4) — 1080×1920, 30fps, H.264 + AAC, exactly 20.000s, 4.4MB.

---

## 1. Creative Approach — One Sentence

A five-act problem→solution story (Hook → Tension → Reveal → Authority → CTA) that opens with a bold accusation ("YOUR ADS ARE LEAKING MONEY."), exposes the diagnostic root causes in a shaking diagnostic grid, resolves with the Sapphire Studios wordmark + service stack, validates with client logos and quantified outcomes (320+ campaigns, 4.8× ROAS), and closes on a tight `sapphirestudios.co` CTA — all as pure code, no timeline editor.

## 2. How to Reproduce

```bash
# Install deps
# Requires: Node.js >= 22, FFmpeg

cd sapphire-promo

# Live preview in the browser (hot reload)
npx hyperframes preview

# Lint the composition
npx hyperframes lint

# Render the final MP4
npx hyperframes render --quality high --fps 30 -o video.mp4
```

`index.html` is a thin shell — HTML body structure + `<link>` tags for each CSS file + `<script src>` tags for each act module + a 16-line inline orchestrator that calls each act's `window.__sapphire.animate*(tl)` function on the master GSAP timeline. Every act is still a `<div class="scene clip">` with `data-start` / `data-duration` / `data-track-index` — Hyperframes gates visibility automatically and drives the single `window.__timelines["main"]` GSAP timeline deterministically via `beginFrame` for seekable, reproducible renders.

| 1 — Hook | 0.0–3.5s | "YOUR ADS ARE LEAKING MONEY." | Word-by-word masked slide-up (`y: 150 → 0`), sapphire radial pulse, accent scale-punch, wipe exit via `y` on a full-bleed cover |

| 3 — Reveal | 7.0–12.5s | "SAPPHIRE STUDIOS" + 4 service pills | Radial sapphire orb breath, wordmark slide-up, divider width animation (0 → 920px), 4 service pills cascade, word-stagger tagline reveal |

## 6. Technical Decisions & Gotchas

- **Single-file composition over sub-compositions** — My first attempt used 5 sub-composition HTML files via `data-composition-src`. In the local render pipeline, sub-compositions without per-file visibility gating caused all five scenes to render simultaneously (CTA stacked on top visually dominated frames at t=1.5s). Inlining everything into `index.html` with `class="clip"` + `data-start` + `data-duration` per scene wrapper gave clean per-act visibility gating on the first render.
- **GSAP's supported deterministic properties** are `opacity, x, y, scale, scaleX, scaleY, rotation, width, height, visibility`. I stuck to these for all primary motion; `stroke-dashoffset` on the SVG X is animated via `attr` (works but kept simple).
- **Transform+GSAP conflict** — CSS `transform: translate(-50%, -50%)` plus a GSAP `y` or `scale` tween corrupts the transform matrix (GSAP rewrites the whole `transform` and loses the percentage translate). Fix: center with **flexbox or negative-margin positioning** on the parent, then use GSAP `y` / `scale` safely. All five backgrounds (`#ambient-orb`, `.hook-radial`, `.orb-bg`) were switched off `translate(-50%, -50%)` to margin-based centering for this reason.
- **No non-determinism** — no `Math.random`, no `Date.now`, no fetches — required by the `beginFrame` capture engine.
- **Audio — layered approach** — The soundtrack is built from four separate assets generated by `gen_audio.py` (Python stdlib + FFmpeg, no external deps):
  - `music.mp3`: Punchy 120-BPM promo track using deterministic synthesis — phase-correct frequency-swept kick drum (`phase = 2π·(A·(1−e^{−kt}) + f_low·t)`), 16th-note hi-hats (LCG noise × decay envelope), snare on beats 2&4, 55 Hz bass, Am chord pad entering at 7s, authority shimmer from 12.5s, CTA sizzle from 16.5s, master fade-out 17.5–20s. Processed via `highpass f=25 → compressor 4:1 → normalize`.
  - `sfx-impact.mp3` (0.34s): Freq-swept thud (150→60 Hz) + transient noise click + 1kHz ping. Used 6× for word reveals, scene entries, and the flash.
  - `sfx-whoosh.mp3` (0.74s): Rising-pitch noise sweep (200→3000 Hz²) with attack-decay envelope. Fires at 2.88s, peaks through the wipe-exit at 3.1s.
  - `sfx-stab.mp3` (1.82s): Am-chord orchestral stab (A2+A3+C4+E4+A4+C5) with sub-bass boom and fast attack. Fires at 7.0s to underscore the brand reveal.
  - All SFX are wired as `<audio class="clip">` elements with precise `data-start` / `data-duration`, escalating `data-volume` (0.55 → 1.00 across the four word hits), and `data-track-index="11"` so they layer cleanly above the main music track.

## 7. Agentic Workflow — Claude Code Prompt Chain

This is the actual sequence of prompts and corrections used when driving Claude Code agentically to build this video. Each prompt produced a file change, which was then rendered via `npx hyperframes render --quality draft`, inspected via `ffmpeg -ss <t> -vframes 1` frame extraction, and iterated on.

### Round 1 — Research & planning
> *"I've been assigned a take-home test for an AI Video Engineer role. Build a 20-second 9:16 promotional video for Sapphire Studios (https://sapphirestudios.co/) — a performance creative agency — using HeyGen Hyperframes + Claude Code + GSAP. The video must be 1080×1920, 20.0s exactly, rendered as MP4. Evaluate criteria prioritizes video quality & polish (45%), creativity & storytelling (30%), agentic workflow evidence (20%), and technical implementation (5%). Research Hyperframes, design the creative, and plan the implementation."*

Action taken: launched parallel `Explore` agents to map the project dir (empty) and research the Hyperframes framework, then a `Plan` agent to design the 5-act arc, brand system, GSAP timeline, file structure, and scoring strategy.

### Round 2 — Project init & assets
> *"Initialize a Hyperframes project at sapphire-promo/ using the blank example. Create assets/ with brand-tokens.css (sapphire-core #1A5CFF, midnight #050B18, Syne/Inter fonts), logo-mark.svg (geometric diamond made of two sapphire triangles + white center circle), and music.mp3 (20s cinematic ambient bed — use ffmpeg layered sine synthesis with drones entering at 0s, 3.5s, 7s, 12.5s, 16.5s to match act transitions; fade out 18–20s)."*

### Round 3 — Build each act as a separate composition
> *"Create compositions/01-hook.html (3.5s): 'YOUR ADS ARE LEAKING MONEY.' hook with word-by-word SplitText slide-up, accent word scale-punch, sapphire radial pulse, clip-path wipe exit. Register the timeline as window.__timelines['hook']. Follow all Hyperframes data-attribute rules from CLAUDE.md."*

> *"Create compositions/02-tension.html (3.5s): 3×3 grid of 9 bad-creative diagnostic panels (Hook:MISSING, CTR:0.4%, Audience:WRONG, ROAS:0.8×, Creative:STALE, CTA:ABSENT, Format:GENERIC, Budget:BURNING, Results:NONE). Panels scale+fade in with a deterministic stagger (no Math.random — we need deterministic rendering). Grid shakes via x oscillation. An SVG red X draws across via stroke-dashoffset. Flash white at 3.15s for a cut."*

> *"Create compositions/03-reveal.html (5.5s): Kicker 'INTRODUCING' → SAPPHIRE STUDIOS wordmark (word slide-up) → glowing divider draws in → 4 service pills reveal from below with cascade (UGC/PGC, AIGC+CGI, ANIMATIONS, DATA-BACKED ADS) → pills pulse confidence-scale → tagline 'The creative engine that converts.'"*

> *"Create compositions/04-authority.html (4s): Glass-morphism panel slides up containing 'TRUSTED PERFORMANCE' kicker, 'The numbers speak loudly.' heading, a horizontal client ticker (AMAZON · NIKE · WALMART · DYSON · +47 BRANDS) scrolling left via GSAP, and two stat cards with live count-up to 320+ (campaigns) and 4.8× (ROAS) using onUpdate callbacks. Panel exits with a sapphire gradient sweep."*

> *"Create compositions/05-cta.html (3.5s): Final CTA with the SVG diamond logo scaling in, two rotating concentric rings, SAPPHIRE STUDIOS wordmark, tagline 'Performance creative, backed by data.', and a rounded CTA pill reading 'sapphirestudios.co →' that pulses with a scale yoyo. Add corner accent marks and a 'BUILT WITH CODE · RENDERED IN HYPERFRAMES' footer."*

### Round 4 — Wire the root composition
> *"Wire all 5 sub-compositions into index.html using data-composition-src + data-composition-id on each host div with correct data-start / data-duration (0/3.5, 3.5/3.5, 7/5.5, 12.5/4, 16.5/3.5). Add the audio <audio id='bg-music'> with data-volume=0.6. Create a 20s ambient background orb that slowly rotates 360° via GSAP. Add a subtle scanline noise overlay on track-index 9."*

### Round 5 — Fix lint errors
> *"`npx hyperframes lint` reported 6 errors: host_missing_composition_id on all 5 sub-composition hosts, and media_missing_id on the audio element. Add data-composition-id='host-hook' etc. to each host, and id='bg-music' to the audio. Also fix the #ambient-orb overlapping scale/rotation tweens warning by removing the redundant breathing scale animation."*

### Round 6 — Critical debug: scene-gating didn't work
After first render: *"At t=1.5s and t=3.0s the rendered video shows Act 5 CTA content instead of the Act 1 Hook. The data-composition-src hosts aren't gating visibility — all 5 compositions stack and the last in DOM order wins. Inline all 5 acts directly into index.html using `<div class='scene clip' data-start=X data-duration=Y>` wrappers so the Hyperframes `class='clip'` visibility lifecycle works correctly. Put all GSAP timelines into the single `window.__timelines['main']` with time offsets (T0, R0, A0, C0 constants for each act's start)."*

This was the key architectural fix. After re-rendering, scenes appeared at the correct times.

### Round 7 — Fix authority panel positioning
After second render: *"The Act 4 authority panel is rendering in the lower-half of the screen, not centered. GSAP's `y` tween conflicts with the CSS `transform: translate(-50%, -50%)` centering — GSAP overwrites the whole transform matrix. Switch the scene to `display: flex; justify-content: center; align-items: center` so the panel is centered by flexbox, then use `gsap.fromTo(..., { y: 140 }, { y: 0 })` for the slide-up. Apply the same fix to #ambient-orb, .hook-radial, and .orb-bg by replacing translate(-50%, -50%) with explicit margin-based positioning."*

### Round 8 — Final render
> *"Run `npx hyperframes render --quality high --fps 30 -o video.mp4`. Verify with `ffprobe`: 1080×1920, 30fps, 20.000s duration, H.264 + AAC."*

### Round 9 — Documentation
> *"Write README.md that explains the creative approach, the act breakdown, technical gotchas encountered (transform+GSAP conflicts, sub-composition visibility gating), and documents this full prompt chain as evidence of the agentic workflow."*

### Round 10 — Audio overhaul: punchy promo track + synchronized SFX
> *"The current audio is not very interactive. Replace the ambient drone with proper promotional music and add sound effects synchronized to the animations. Music must sound interesting as a promotion brand video."*

Action taken: wrote `gen_audio.py` (Python stdlib synthesis + FFmpeg post-processing) to generate four audio assets. Main track uses a phase-correct kick drum formula, LCG-seeded hi-hat/snare noise, act-synced chord pads and shimmer layers. Three SFX files (impact, whoosh, stab) added as timed `<audio class="clip">` elements. Escalating `data-volume` on the 4 word-reveal hits creates a building dramatic arc. `npx hyperframes lint` still passes with 0 errors.

### Round 11 — Code refactor: split monolithic index.html into per-act files
> *"The index.html file is flooded with code. Refactor it into multiple files more carefully than the first attempt — which used data-composition-src sub-compositions and caused all five scenes to render simultaneously."*

Action taken: instead of retrying sub-compositions (which carry per-file lifecycle issues), extracted only the CSS and JS into external files — leaving the proven `class="clip"` scene-gating HTML structure intact in `index.html`. Six CSS files (`styles/base.css`, `styles/act1-hook.css` … `styles/act5-cta.css`) and six JS modules (`scripts/acts/ambient.js`, `01-hook.js` … `05-cta.js`) were created. Each JS module attaches a `window.__sapphire.animateXxx(tl)` function; a 16-line inline orchestrator in `index.html` creates the master GSAP timeline, calls each act in order, and registers it on `window.__timelines["main"]`. The orchestrator stays inline because the Hyperframes linter does a static text scan of `index.html` for the `window.__timelines` registration. `index.html` shrank from 1187 → 275 lines. `npx hyperframes lint` passes with 0 errors; a draft render frame-sampled across all five acts was visually confirmed to match the baseline `video.mp4`.

### Iterations summary

- **2 full draft renders** before the final high-quality render (first to discover the sub-composition gating bug, second to validate the inlined-single-file rewrite).
- **1 architectural rewrite** (5 sub-composition files → 1 single-file composition) driven by frame inspection.
- **1 CSS refactor** across 4 elements to decouple CSS centering transforms from GSAP motion.
- **1 code-organisation refactor** (monolithic `index.html` → thin shell + `styles/` + `scripts/acts/`) after the initial sub-composition attempt failed; used external `<link>`/`<script>` files instead to avoid the lifecycle issue.
- **Continuous verification** by extracting individual frames at act-boundary timestamps with `ffmpeg -ss <t> -vframes 1` and visually inspecting them — much faster than watching the full video.
