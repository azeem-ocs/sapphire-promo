# Workflow Summary

I used the job description and the test task brief to define the requirements, creative direction, and delivery constraints for this assignment.

I first used Claude Sonnet 4.6 to turn that brief into a structured implementation plan covering the concept, scene flow, technical approach, and execution steps.

That planning document is captured here:

- [sonnet-4-6-plan.md](./sonnet-4-6-plan.md)

After the planning phase, I switched to Claude Opus 4.7 as the primary implementation model and used it to build the project against that plan.

I then reviewed the output, iterated on the implementation, and documented the build process, technical decisions, debugging steps, and final delivery details in:

- [README.md](./README.md)

In short, the workflow was:

1. Review the brief and extract requirements.
2. Generate a structured implementation plan with Sonnet 4.6.
3. Execute the build with Opus 4.7.
4. Review, refine, and document the final result.
5. Audio overhaul: replaced the ambient drone with a punchy 120-BPM promotional track and added 9 synchronized SFX cues (word-reveal impacts, transition whoosh, brand-reveal stab, scene-entry hits) — see Round 10 in [README.md](./README.md).
